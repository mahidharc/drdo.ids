Working of the Naive Bayes Classifier

To understand the working of the Naive Bayes Classifier, knowledge of its mathematical representation is very important.

Let us consider three sets, 'X', 'D' and 'C'.

Let Set 'X' represent the attribute set or the features set. It consists of attributes x1 through xn. Therefore, the set can be represented as X={x1,x2,x3,....., xn}.

Let Set 'C' represent the class set. It consists of classes c1 through cm. Therefore, the set can be represented as c={c1,c2,c3,....., cm}.

Let Set 'D' represent the Training set. It consists of tuples of the type {xn,cm}. Therefore, the set can be represented as D={{x1, cm},{x2, cm}, {x3,cm},....., {xn,cm}}.

To find out the association of 'X' with a particular class ci, we use the Bayes Theorem and the Independence Assumption.

Using Bayes Theorem we get,

P(ci| X) = P( X| ci) x P( ci)/ P(X)


To simplify the calculation of P( X| ci), we use the Independence assumption. The Independence Assumption states that presence or absence of a particular feature in a class is completely independent of the presence or absence of that feature in any other class.


Through this we get,

P( X| ci) = P(x1| ci) x P(x2| ci) x ... x P(xn| ci)

Replacing this we get, P(ci| X) = P(x1| ci) x P(x2| ci) x ... x P(xn| ci) x P( ci)/ P(X)

The output of the Naive Bayes Classifier is nothing but the maximum of P(ci| X). Through this the Naive Bayes Classifier groups data into anyone of the classes specified.
